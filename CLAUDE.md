## 개발 규칙 - 기능 변경 정책

**🚨 중요: 기능 추가/삭제 금지 원칙**

### 절대 규칙
1. **기능 추가 금지**: 사용자가 명시적으로 요청하지 않은 기능은 절대 추가하지 말 것
2. **기능 삭제 금지**: 사용자가 명시적으로 삭제 요청하지 않은 기능은 절대 제거하지 말 것
3. **기존 기능 유지**: 구현된 모든 기능은 필요에 의해 만들어진 것이므로 보존할 것

### 허용되는 작업
- ✅ 버그 수정
- ✅ 성능 개선  
- ✅ UI/UX 개선
- ✅ 코드 리팩토링
- ✅ 타입 오류 수정
- ✅ 사용자가 명시적으로 요청한 수정사항

### 금지되는 작업
- ❌ 임의로 기능 추가
- ❌ 임의로 기능 삭제
- ❌ 사용자 요청 없는 구조 변경
- ❌ 사용자 요청 없는 UI 컴포넌트 추가/제거

**원칙**: 사용자가 "기능 추가 필요" 또는 "기능 삭제 필요"라고 명시적으로 요청할 때만 해당 작업 수행
