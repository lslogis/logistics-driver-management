**Last Updated**: 2025-09-11  
**Target Audience**: 최종 사용자, 관리자, 디스패처, 회계담당자

## 🎯 시스템 개요

**운송기사관리 시스템**은 운송회사의 기사, 차량, 노선, 운행, 정산을 통합 관리하는 웹 기반 시스템입니다.

### 주요 사용자 그룹
- **관리자 (ADMIN)**: 전체 시스템 관리 권한
- **디스패처 (DISPATCHER)**: 운행 계획 및 기사/차량 관리
- **회계담당자 (ACCOUNTANT)**: 정산 처리 및 회계 업무

### 핵심 기능
1. **기사 관리**: 운송기사 등록, 수정, 비활성화
2. **차량 관리**: 차량 등록, 기사 배정, 소유권 관리
3. **노선 관리**: 고정노선 템플릿, 요일별 운행 패턴
4. **운행 관리**: 일일 운행 등록, 상태 관리, 결행/대차 처리
5. **정산 관리**: 월별 정산 계산, 미리보기, 확정, 내보내기
6. **데이터 임포트**: CSV 파일을 통한 대량 데이터 등록

## 🏠 메인 대시보드

### 접속 방법
1. 웹 브라우저에서 `http://localhost:3000` 접속
2. 로그인 화면에서 사용자 계정으로 로그인
3. 메인 대시보드 진입

### 대시보드 구성
- **상단 네비게이션**: 메인 메뉴, 로그아웃 버튼
- **좌측 사이드바**: 주요 메뉴 (기사관리, 차량관리, 노선관리, 운행관리, 정산관리)
- **메인 콘텐츠**: 통계 카드 및 최근 활동
- **빠른 액션**: 자주 사용하는 기능 바로가기

### 통계 정보
- **총 기사 수**: 활성 기사 현황
- **총 차량 수**: 등록 차량 현황  
- **오늘 운행**: 금일 운행 계획 및 완료 현황
- **이번 달 정산**: 정산 대상 및 완료 현황

## 👥 기사 관리 (/drivers)

### 메뉴 접근: 대시보드 → 기사 관리

### 기사 목록 화면
**화면 구성**:
- **검색 바**: 기사명 또는 전화번호로 검색
- **필터**: 상태별 필터 (활성/비활성)
- **정렬**: 이름, 등록일, 전화번호순 정렬
- **기사 카드**: 기사 정보가 카드 형태로 표시

**기사 카드 정보**:
- **기본 정보**: 이름, 전화번호, 이메일
- **회사 정보**: 사업자등록번호, 회사명, 대표자명
- **계좌 정보**: 은행명, 계좌번호
- **상태**: 활성/비활성 상태 표시
- **액션 버튼**: 수정, 비활성화/활성화, 삭제

### 기사 등록
1. **[+ 기사 등록]** 버튼 클릭
2. **필수 정보 입력**:
   - 이름 (필수)
   - 전화번호 (필수, 중복 불가)
3. **선택 정보 입력**:
   - 이메일
   - 사업자등록번호
   - 회사명, 대표자명
   - 은행명, 계좌번호
   - 비고
4. **[등록]** 버튼으로 저장

### 기사 수정
1. 기사 카드에서 **[수정]** 버튼 클릭
2. 모달창에서 정보 수정
3. **[저장]** 버튼으로 변경사항 저장

### 기사 비활성화
1. 기사 카드에서 **[비활성화]** 버튼 클릭
2. 확인 다이얼로그에서 **[확인]** 클릭
3. 비활성화된 기사는 회색으로 표시됨

**주의사항**:
- 전화번호는 시스템 내에서 고유해야 함
- 비활성화된 기사는 새 운행에 배정 불가
- 기존 운행 기록은 유지됨

## 🚗 차량 관리 (/vehicles)

### 메뉴 접근: 대시보드 → 차량 관리

### 차량 목록 화면
**화면 구성**:
- **검색 바**: 차량번호 또는 차량 유형으로 검색
- **필터**: 소유권별 필터 (고정, 용차, 지입)
- **정렬**: 차량번호, 등록일, 소유권순 정렬
- **차량 카드**: 차량 정보가 카드 형태로 표시

**차량 카드 정보**:
- **기본 정보**: 차량번호, 차량 유형, 소유권
- **배정 정보**: 담당 기사명 (배정 시)
- **부가 정보**: 적재중량, 연식
- **상태**: 활성/비활성 상태 표시
- **액션 버튼**: 수정, 기사 배정, 비활성화, 삭제

### 차량 등록
1. **[+ 차량 등록]** 버튼 클릭
2. **필수 정보 입력**:
   - 차량번호 (필수, 중복 불가)
   - 차량 유형 (예: 1톤, 2.5톤, 5톤)
   - 소유권 (고정/용차/지입)
3. **선택 정보 입력**:
   - 담당 기사 선택
   - 적재중량
   - 연식
4. **[등록]** 버튼으로 저장

### 기사 배정/해제
1. 차량 카드에서 **[기사 배정]** 버튼 클릭
2. 드롭다운에서 기사 선택 또는 "배정 해제" 선택
3. **[저장]** 버튼으로 변경사항 저장

**소유권 유형**:
- **고정**: 회사 소유 차량
- **용차**: 임대 차량
- **지입**: 기사 개인 소유 차량

## 🗺️ 노선 관리 (/routes)

### 메뉴 접근: 대시보드 → 노선 관리

### 노선 템플릿 목록
**화면 구성**:
- **검색 바**: 노선명 또는 출발지/도착지로 검색
- **필터**: 요일별 운행 패턴 필터
- **노선 카드**: 노선 정보가 카드 형태로 표시

**노선 카드 정보**:
- **노선 정보**: 노선명, 출발지 → 도착지
- **요금 정보**: 기사요금, 청구요금
- **운행 패턴**: 요일별 운행 여부 (월화수목금토일)
- **거리**: 편도 거리 (km)
- **기본 기사**: 배정된 기본 기사명
- **액션 버튼**: 수정, 기사 배정, 비활성화

### 노선 등록
1. **[+ 노선 등록]** 버튼 클릭
2. **필수 정보 입력**:
   - 노선명 (필수, 중복 불가)
   - 출발지 (필수)
   - 도착지 (필수)
   - 기사요금 (필수)
   - 청구요금 (필수)
3. **운행 패턴 설정**:
   - 요일별 체크박스로 운행일 선택
   - 월요일(1) ~ 일요일(7) 패턴
4. **선택 정보**:
   - 거리 (km)
   - 기본 기사 배정
5. **[등록]** 버튼으로 저장

### 요일별 운행 패턴
- **패턴 표시**: 월화수목금토일 (1234567)
- **활성 요일**: 파란색 배지로 표시
- **비활성 요일**: 회색 배지로 표시
- **예시**: 월~금 운행 = [1][2][3][4][5]

## 📋 운행 관리 (/trips)

### 메뉴 접근: 대시보드 → 운행 관리

### 운행 목록 화면
**화면 구성**:
- **날짜 필터**: 날짜 범위 또는 특정일 검색
- **상태 필터**: 예정/완료/결행/대차별 필터
- **기사/차량 검색**: 기사명 또는 차량번호로 검색
- **운행 목록**: 테이블 형태로 운행 정보 표시

**운행 목록 정보**:
- **날짜**: 운행 예정일
- **기사**: 담당 기사명 및 연락처
- **차량**: 차량번호 및 유형
- **노선**: 출발지 → 도착지
- **상태**: 예정/완료/결행/대차
- **요금**: 기사요금, 청구요금
- **액션**: 상태 변경, 수정, 삭제

### 운행 등록
1. **[+ 운행 등록]** 버튼 클릭
2. **기본 정보 입력**:
   - 운행일 (필수)
   - 기사 선택 (필수)
   - 차량 선택 (필수)
3. **노선 정보**:
   - 노선 유형: 고정노선/임시노선
   - 고정노선: 템플릿에서 선택
   - 임시노선: 출발지/도착지 직접 입력
4. **요금 정보**:
   - 기사요금 (자동 입력 또는 수정 가능)
   - 청구요금 (자동 입력 또는 수정 가능)
5. **[등록]** 버튼으로 저장

### 운행 상태 관리
**상태 유형**:
1. **예정 (SCHEDULED)**: 운행 계획 등록 상태
2. **완료 (COMPLETED)**: 정상 운행 완료
3. **결행 (ABSENCE)**: 기사 사정으로 운행 불가
4. **대차 (SUBSTITUTE)**: 다른 기사가 대신 운행

**상태 변경 방법**:
1. 운행 목록에서 **[상태 변경]** 버튼 클릭
2. 새 상태 선택
3. **결행/대차 시 추가 정보**:
   - 결행 사유 입력
   - 공제액 설정 (기본: 결행 10%, 대차 5%)
   - 대차 시 대체 기사 선택

### 운행 완료 처리
1. 운행 목록에서 **[완료 처리]** 버튼 클릭
2. 완료 확인 다이얼로그에서 **[확인]** 클릭
3. 상태가 "완료"로 변경됨
4. 해당 운행은 정산 대상에 포함됨

## 💰 정산 관리 (/settlements)

### 메뉴 접근: 대시보드 → 정산 관리

### 정산 목록 화면
**화면 구성**:
- **검색 바**: 기사명으로 검색
- **월 필터**: 정산월별 필터 (YYYY-MM 형식)
- **상태 필터**: 임시저장/확정/지급완료별 필터
- **정산 목록**: 기사별 정산 정보 카드 표시

**정산 카드 정보**:
- **기사 정보**: 기사명, 연락처, 정산월
- **정산 요약**: 총 운행 횟수, 기본요금, 공제액, 최종 정산액
- **상태**: 임시저장/확정/지급완료 배지
- **일시**: 생성일, 확정일 (확정 시)
- **액션**: 미리보기, 확정, 내보내기

### 빠른 정산 처리
**화면 상단 섹션**:
1. **정산월 선택**: 월 입력 (YYYY-MM)
2. **기사 ID 입력**: 특정 기사 또는 전체
3. **[정산 미리보기 → 확정]** 버튼 클릭

**처리 과정**:
1. 입력한 월의 해당 기사 운행 데이터 조회
2. 자동 정산 계산 수행
3. 미리보기 모달 표시
4. 확인 후 정산 확정 처리

### 정산 미리보기
**미리보기 모달 내용**:
- **정산 요약**: 총 운행, 기본요금, 공제액, 최종 정산액
- **주의사항**: 계산 과정에서 발견된 이슈 표시
- **운행 내역 테이블**: 
  - 날짜, 상태, 노선, 기사요금, 차감액
  - 최대 50건 표시 (스크롤 가능)
- **액션**: 취소 또는 정산 확정

### 정산 확정 (월락)
1. 미리보기에서 **[정산 확정 (월락)]** 버튼 클릭
2. 확인 다이얼로그에서 **[확인]** 클릭
3. 정산 상태가 "확정"으로 변경
4. **주의**: 확정 후에는 수정 불가

### 정산 내보내기
1. 정산 카드에서 **[내보내기]** 버튼 클릭
2. 또는 월 필터 선택 후 **[엑셀 내보내기]** 버튼
3. 정산 내역이 엑셀 파일로 다운로드

**내보내기 포함 정보**:
- 기사 기본 정보
- 운행별 상세 내역
- 공제 내역
- 최종 정산 금액

## 📥 데이터 임포트

### 기사 CSV 임포트 (/import/drivers)

**메뉴 접근**: 기사 관리 → CSV 가져오기

**임포트 과정**:
1. **템플릿 다운로드**:
   - **[템플릿 다운로드]** 버튼 클릭
   - 기사 CSV 템플릿 파일 다운로드
2. **CSV 파일 준비**:
   - 템플릿에 맞춰 기사 정보 입력
   - 인코딩: UTF-8 권장
   - 최대 파일 크기: 10MB
3. **파일 업로드**:
   - 드래그 앤 드롭 또는 클릭하여 파일 선택
   - CSV 파일만 업로드 가능
4. **검증 및 미리보기**:
   - **[검증 및 미리보기]** 버튼 클릭
   - 데이터 유효성 검사 수행
   - 처음 5개 레코드 미리보기 표시
   - 오류 내역 확인
5. **최종 등록**:
   - 검증 완료 후 **[N개 기사 등록]** 버튼 활성화
   - 클릭하여 실제 등록 수행

**CSV 필수 컬럼**:
- **name**: 기사명 (필수)
- **phone**: 전화번호 (필수, 중복 불가)

**CSV 선택 컬럼**:
- email, businessNumber, companyName, representativeName
- bankName, accountNumber, remarks

### 운행 CSV 임포트 (/import/trips)

**메뉴 접근**: 운행 관리 → CSV 가져오기

**임포트 과정**: 기사 임포트와 동일한 절차

**CSV 필수 컬럼**:
- **date**: 운행일 (YYYY-MM-DD)
- **driverId**: 기사 ID
- **vehicleId**: 차량 ID
- **driverFare**: 기사요금
- **billingFare**: 청구요금

**주의사항**:
- 동일 날짜, 기사, 차량 조합은 중복 불가
- 존재하지 않는 기사/차량 ID는 오류 처리
- 잘못된 날짜 형식은 오류 처리

## ⚙️ 시스템 설정 및 관리

### 사용자 권한별 접근 제한

**관리자 (ADMIN)**:
- 모든 기능 접근 가능
- 사용자 관리, 시스템 설정

## 🚨 일반적인 문제 해결

### 로그인 문제
**증상**: 로그인이 되지 않음  
**해결책**: 
1. 이메일과 비밀번호 확인
2. 브라우저 쿠키 삭제 후 재시도
3. 관리자에게 계정 상태 확인 요청

### CSV 임포트 오류
**증상**: CSV 업로드 시 오류 발생  
**해결책**:
1. 템플릿 파일과 형식 일치 확인
2. 파일 인코딩 UTF-8 확인
3. 필수 컬럼 누락 확인
4. 중복 데이터 확인

### 정산 계산 오류
**증상**: 정산 금액이 예상과 다름  
**해결책**:
1. 해당 월 운행 기록 확인
2. 결행/대차 공제율 확인
3. 기사요금 설정 확인
4. 관리자에게 문의

### 성능 저하
**증상**: 페이지 로딩이 느림  
**해결책**:
1. 검색 조건을 구체적으로 설정
2. 날짜 범위를 축소하여 조회
3. 브라우저 캐시 삭제
4. 네트워크 연결 상태 확인
